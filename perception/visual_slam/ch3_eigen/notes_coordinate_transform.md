# ch3 笔记:坐标系之间的点变换

> 对应 slambook2 习题 `ch3/examples/coordinateTransform.cpp`

## 一、题目

两台机器人各有一个位姿(相对世界系):

| 机器人 | 旋转(四元数) | 平移 | 位姿矩阵 |
|---|---|---|---|
| 1 号 | `q1` | `t1` | `T1w` |
| 2 号 | `q2` | `t2` | `T2w` |

`T1w`、`T2w` 都是 **"世界系 → 机器人系"** 的变换。

1 号机器人在**自己的坐标系**里看到一个点 `p1 = (0.5, 0, 0.2)`。
**求:该点在 2 号机器人坐标系下的坐标 `p2`。**

## 二、核心代码

```cpp
Quaterniond q1(0.35, 0.2, 0.3, 0.1), q2(-0.5, 0.4, -0.1, 0.2);
q1.normalize();                       // 必须归一化
q2.normalize();
Vector3d t1(0.3, 0.1, 0.1), t2(-0.1, 0.5, 0.3);
Vector3d p1(0.5, 0, 0.2);

Isometry3d T1w(q1), T2w(q2);          // 用四元数构造,先只有旋转
T1w.pretranslate(t1);                 // 再补平移
T2w.pretranslate(t2);

Vector3d p2 = T2w * T1w.inverse() * p1;
```

运行结果:`p2 ≈ (-0.0309, 0.7404, 1.0815)`

## 三、变换链怎么读

`p2 = T2w * T1w.inverse() * p1` —— 矩阵乘法**从右往左**依次作用:

```
p1  ──T1w.inverse()──▶  p_world  ──T2w──▶  p2
(1号系)                  (世界系)            (2号系)
```

1. **`T1w.inverse()`**:`T1w` 是"世界→1号",取逆得到"1号→世界"。
   作用在 `p1` 上 → 把点从 **1号系** 换到 **世界系**。
2. **`T2w`**:本身是"世界→2号"。
   作用在上一步结果上 → 把点从 **世界系** 换到 **2号系**。

以世界系作"桥梁":`1号系 → 世界系 → 2号系`。

## 四、两个关键要点

### 1. 乘法顺序 = 作用顺序的逆序

代码里 `T2w` 写在最左,但它**最后**才作用;`p1` 在最右,**最先**被处理。
链式变换永远"离点最近的先算"。

### 2. 下标接龙法 —— 检查变换链有没有写反

把变换写成 `下标A←下标B` 的形式:

- `T2w` 读作 `2 ← w`
- `T1w.inverse()` = `Tw1`,读作 `w ← 1`

拼起来:`T2w · Tw1 · p1` → 下标 `2 ← w ← 1`,中间的 `w` 抵消,
左右剩 `2 ← 1`,即"1号系的点 → 2号系的点" ✅。

**下标能不能接龙对消,是判断变换链写没写反的最快方法。**

## 五、易错点

- **四元数必须 `.normalize()`**:题目给的不是单位四元数,不归一化无法表示合法旋转。
- **构造顺序**:`Quaterniond(w, x, y, z)` —— w 实部在**前**;
  而 `q.coeffs()` 读出来是 `(x, y, z, w)` —— w 在**后**。两者顺序相反。
- **`Isometry3d` 构造**:`Isometry3d T(q)` 只设了旋转,平移要再 `.pretranslate(t)` 补。
- **`Isometry3d` 实质是 4×4 齐次矩阵**,`T * 三维点` 等价于 `R * 点 + t`。

## 六、工程意义

这是 SLAM 中最基础也最高频的操作:**已知两坐标系各自相对世界系的位姿,
把一个点从一个坐标系换算到另一个**。

典型应用:把上一帧相机看到的特征点,变换到当前帧相机坐标系下做匹配。
