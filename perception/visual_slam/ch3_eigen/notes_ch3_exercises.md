## 习题 2:罗德里格斯公式的推导

### 目标

已知旋转轴(单位向量)`n` 和旋转角 `θ`,求对应的旋转矩阵 `R`。
最终结论:

```
R = cosθ · I + (1 - cosθ) · n·nᵀ + sinθ · n^
```

其中 `n^` 是 `n` 的反对称矩阵(叉乘矩阵):

```
n^ = [  0   -n_z   n_y ]
     [  n_z   0   -n_x ]
     [ -n_y   n_x   0  ]
```

### 推导思路

**第一步:把向量 v 沿旋转轴分解**

把待旋转向量 `v` 拆成两部分:

- **平行分量** `v∥`:沿轴 `n` 方向的投影 → `v∥ = (n·v) n = (n nᵀ) v`
- **垂直分量** `v⊥`:`v⊥ = v - v∥ = (I - n nᵀ) v`

旋转只改变垂直分量,平行分量不动(绕轴转,轴上的部分不变)。

**第二步:在垂直平面内旋转 v⊥**

在垂直于 `n` 的平面里,取两个正交基:

- `v⊥` 本身
- `w = n × v`(等于 `n^ v`),它与 `v⊥` 等长且垂直

垂直分量旋转 `θ` 后:

```
v⊥' = cosθ · v⊥ + sinθ · w
    = cosθ · v⊥ + sinθ · (n^ v)
```

(这就是二维平面内"绕原点转 θ"的标准分解。)

**第三步:合并**

```
v' = v∥ + v⊥'
   = (n nᵀ) v  +  cosθ (I - n nᵀ) v  +  sinθ (n^) v
```

提取出矩阵 `R`(满足 `v' = R v`):

```
R = n nᵀ + cosθ (I - n nᵀ) + sinθ · n^
  = cosθ · I + (1 - cosθ) · n nᵀ + sinθ · n^
```

得证。

### 关键理解

| 项 | 几何意义 |
|---|---|
| `cosθ · I` | 主体旋转,角度越大保留越少 |
| `(1-cosθ) · n nᵀ` | 把"沿轴方向"的分量补回来(轴上不旋转) |
| `sinθ · n^` | 垂直平面内的"叉乘方向"分量,提供旋转的偏转 |

- `n nᵀ` 是 **投影矩阵**(投到轴方向),`I - n nᵀ` 投到垂直平面。
- `n^ v = n × v`:反对称矩阵就是"叉乘"的矩阵写法。
- 验证:`θ = 0` 时 `R = I`(不转);`n nᵀ` 部分保证轴方向永远不变。

## 习题 4:四种旋转表示的转换关系

四种表示:**旋转矩阵 R**、**轴角(旋转向量)**、**欧拉角**、**四元数 q**。

### 转换关系表(含 Eigen API)

| 从 \ 到 | 旋转矩阵 R | 轴角 AngleAxis | 欧拉角 | 四元数 Quaternion |
|---|---|---|---|---|
| **旋转矩阵 R** | — | `AngleAxisd(R)` | `R.eulerAngles(2,1,0)` ¹ | `Quaterniond(R)` |
| **轴角** | `aa.toRotationMatrix()` 或 `aa.matrix()` | — | 先转 R 再转 | `Quaterniond(aa)` |
| **欧拉角** | 三个 `AngleAxisd` 连乘 ² | 先转 R 再转 | — | 三个 `AngleAxisd` 连乘 → q |
| **四元数 q** | `q.toRotationMatrix()` 或 `q.matrix()` | `AngleAxisd(q)` | 先转 R 再转 | — |

¹ 新版 Eigen 用 `R.canonicalEulerAngles(2,1,0)`,旧名 `eulerAngles` 已弃用。
² 例:`AngleAxisd(yaw,Z) * AngleAxisd(pitch,Y) * AngleAxisd(roll,X)`。

### 要点

- **R / 轴角 / 四元数可以直接互转**,Eigen 都提供了构造函数或成员函数。
- **欧拉角没有直接的互转构造函数** —— 通常以旋转矩阵 `R` 作中转。
- 欧拉角 → R:按约定顺序(如 ZYX)把三个 `AngleAxisd` **连乘**。
- R → 欧拉角:`canonicalEulerAngles(轴0,轴1,轴2)`,`(2,1,0)` = ZYX = yaw-pitch-roll。

### 四种表示的特点对比

| 表示 | 参数个数 | 优点 | 缺点 |
|---|---|---|---|
| 旋转矩阵 R | 9(3×3) | 直接用于运算、变换点 | 冗余多,9 个数表示 3 自由度 |
| 轴角 | 4(角+轴) | 直观,物理意义清晰 | 不便于插值、叠加旋转 |
| 欧拉角 | 3 | 人最易理解(yaw-pitch-roll) | **万向锁**;不唯一 |
| 四元数 q | 4 | 紧凑、无奇异、易插值(SLERP) | 不直观;有正负二义性(q 与 -q 同旋转) |

### 工程选择

- **存储 / 插值** → 四元数
- **计算位姿、变换点** → 旋转矩阵 / 变换矩阵 `Isometry3d`
- **人看 / 调试** → 欧拉角

> 万向锁:欧拉角中间那个轴转到 ±90° 时,另两个轴重合,丢失一个自由度。
> 对应练习见 `ex03_gimbal_lock.cpp`。

## 习题 1:验证旋转矩阵是正交矩阵

旋转矩阵的两个判据:

- **正交性**:`R · Rᵀ = I`
- **行列式**:`det(R) = 1`(若为 -1 则是"旋转 + 镜像",不是纯旋转)

```cpp
Matrix3d R = AngleAxisd(M_PI/4, Vector3d(1,0,0)).toRotationMatrix();
cout << R * R.transpose() << "\n";   // ≈ 单位阵(对角线外有 ~1e-17 浮点误差)
cout << R.determinant() << "\n";     // ≈ 1
```

- 浮点运算不会得到精确的 0/1,判等用 `isApprox()` 或看 `.norm()` 是否够小。
- `Matrix3d` 只是 3×3 容器,要用 `AngleAxisd().toRotationMatrix()` 才能得到**合法**旋转矩阵。

## 习题 3:四元数旋转点 → 结果仍是虚四元数

把三维点 `p=(x,y,z)` 表示成**纯(虚)四元数** `p_quat = (0, x, y, z)`(实部 w=0)。

旋转公式:`p' = q · p_quat · q⁻¹`

```cpp
Quaterniond q(...);  q.normalize();          // 旋转四元数必须是单位四元数
Quaterniond p_quat(0, 1, 0, 0);              // 点 (1,0,0) → 纯四元数
Quaterniond p_rotated = q * p_quat * q.inverse();
// p_rotated.w() ≈ 0  → 结果仍是纯四元数,对应一个三维点
// p_rotated.vec() 即旋转后的点;模长不变(旋转保持长度)
```

结论:旋转后实部恒为 0,所以结果**仍能解释成一个三维空间点** —— 这就是用四元数旋转向量的理论依据。

- `Quaterniond` 不能写成 `Quaternion`(模板必须带类型参数)。
- `.vec()` 取虚部 (x,y,z);`.w()` 取实部。

## 习题 5:取矩阵的块并赋值

```cpp
MatrixXd big = MatrixXd::Random(8, 8);
big.block<3,3>(0, 0) = Matrix3d::Identity();   // 左上角 3×3 块赋值为 I
```

块操作要点:

- `.block<行,列>(起始行, 起始列)` —— 编译期已知尺寸,效率高
- `.block(i, j, p, q)` —— 运行期尺寸
- `.topLeftCorner<3,3>()` —— 取左上角的专用简写
- 块表达式是矩阵的**引用视图**,**可读可写** —— 放在 `=` 左边就是写入原矩阵。

## 习题 6:求解 A x = b 的三种方法

由 `A x = b`,两边左乘 `A⁻¹` 得 `x = A⁻¹ b`(逆在左、b 在右,顺序不能反)。

| 方法 | Eigen 写法 | 适用条件 | 速度 | 稳定性 |
|---|---|---|---|---|
| 直接求逆 | `A.inverse() * b` | 方阵可逆 | 慢 | 差 |
| QR 分解 | `A.colPivHouseholderQr().solve(b)` | 几乎无限制 | 中 | 好 |
| Cholesky | `A.ldlt().solve(b)` | **对称正定** | 快 | 好 |

- 三种方法数学上解相同,差别只在**效率和数值精度**;对比 `(x₂-x₁).norm()` 应 ≈ 0。
- `A = A * A.transpose()` 的作用:任意矩阵乘自身转置,结果必**对称半正定**,这样 `ldlt()` 才可用。
  再 `+ I` 可保证**正定且良态**,避免病态矩阵把解放大。
- **选择优先级**:对称正定 → Cholesky(最快);不确定矩阵性质 → QR(最通用);
  求逆基本不用来解方程(只在确实需要逆矩阵本身时用)。
- 工程意义:SLAM 优化问题(BA、位姿图)的矩阵通常对称正定,故 Cholesky 用得最多。
